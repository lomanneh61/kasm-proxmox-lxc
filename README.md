Kasm Workspaces on Proxmox LXCA complete automation suite for installing, updating, and managing Kasm Workspaces inside a Debian 12 privileged LXC container on Proxmox VE.This project provides:One‑click / one‑command installerOne‑command uninstallerAuto‑update script with Email, Discord, or Telegram notificationsOptional systemd timer for nightly updatesOptional Proxmox GUI button for one‑click updatesClean, reproducible LXC configuration compatible with Docker and KasmOne‑Click InstallationInstall Kasm with a single command directly from your Proxmox host:Codebash <(curl -s https://raw.githubusercontent.com/lomanneh61/kasm-proxmox-lxc/main/scripts/kasm-lxc-installer.sh) $(pvesh get /cluster/nextid)This automatically:Selects the next available CTIDCreates a Debian 12 privileged LXCApplies all required Kasm/Docker LXC settingsInstalls DockerInstalls Kasm WorkspacesPerforms validation checksManual InstallationKasm can be installed into any CTID you choose.Install into a specific CTIDCodebash scripts/kasm-lxc-installer.sh <CTID>Example:Codebash scripts/kasm-lxc-installer.sh 200Automatically use the next available CTIDCodeNEXT=$(pvesh get /cluster/nextid)bash scripts/kasm-lxc-installer.sh $NEXTThe installer will:Create a Debian 12 privileged LXCApply all required Kasm/Docker LXC settingsInstall DockerInstall Kasm WorkspacesValidate device passthrough (/dev/fuse, /dev/net/tun)Updating KasmThe update script automatically detects:Installed Kasm versionLatest available Kasm releaseWhether an upgrade is requiredUpdate with no notificationsCodebash scripts/kasm-lxc-auto-update.sh --notify-noneUpdate with Email notificationCodebash scripts/kasm-lxc-auto-update.sh --notify-email you@example.comUpdate with Discord webhookCodebash scripts/kasm-lxc-auto-update.sh --notify-discord https://discord.com/api/webhooks/XXXX/YYYYUpdate with Telegram notificationCodebash scripts/kasm-lxc-auto-update.sh --notify-telegram BOT\_TOKEN:CHAT\_IDThe updater performs:Graceful shutdown of Kasm containersBackup of /opt/kasmDownload and extraction of the latest releaseIn‑place upgradeCleanup and health checksUninstalling KasmTo remove Kasm completely from a container:Codebash scripts/kasm-lxc-uninstall.shThis removes:All Kasm containersAll Kasm Docker imagesKasm networks and volumesrclone plugin/opt/kasm and related directoriesOptional: Nightly Auto‑Update (systemd)Enable nightly updates at 03:00:Codecp systemd/kasm-update.service /etc/systemd/system/cp systemd/kasm-update.timer /etc/systemd/system/systemctl enable --now kasm-update.timerEdit the service file to set your preferred CTID and notification method.Optional: Proxmox GUI IntegrationA one‑click “Update Kasm” button can be added to the Proxmox LXC menu.Instructions are in:Codeproxmox-gui/kasm-update-menu.mdThis integrates with the wrapper script:Codescripts/kasm-lxc-gui-wrapper.shNotificationsDocumentation for each notification method is
